# Configuration

The configuration surfaces in My SPACE: the Chrome MV3 manifest, the Vite build-time config, the `chrome.storage.local` keys the extension reads and writes at runtime, the Android `SharedPreferences` used by the sync layer, and the OAuth scopes shared by both platforms.

## Chrome extension manifest

The manifest lives at `chrome-extension/manifest.json` and is the source of truth for the extension's identity, permissions, and entry points. The `@crxjs/vite-plugin` reads it at build time (see [Tooling](../how-to-contribute/tooling.md)).

### Identity and version

| Field | Value | Notes |
|---|---|---|
| `manifest_version` | `3` | MV3. |
| `name` | `"My SPACE"` | Display name. |
| `version` | `"0.3.0"` | Bumped in lockstep with `package.json` by `scripts/bump.js`. |
| `description` | `"Free vault for your space"` | Web store description. |
| `key` | (redacted) | Extension key used to pin the extension ID across loads. |

### Permissions

```json
"permissions": ["storage", "offscreen", "identity", "sidePanel", "activeTab"]
```

| Permission | Why |
|---|---|
| `storage` | `chrome.storage.local` for vault salt, lock timeout, and Drive sync state (see below). |
| `offscreen` | The offscreen document that hosts PGlite and Web Crypto. |
| `identity` | `chrome.identity.launchWebAuthFlow` for Google OAuth. |
| `sidePanel` | The side panel UI. |
| `activeTab` | Access to the current tab context when the action is invoked. |

### Host permissions

```json
"host_permissions": ["https://www.googleapis.com/*"]
```

Required for Drive API and Google userinfo API calls from the service worker.

### Entry points

| Field | Path | Purpose |
|---|---|---|
| `background.service_worker` | `src/service-worker/index.ts` | Drive sync, OAuth, message relay. Module type. |
| `side_panel.default_path` | `src/sidepanel/index.html` | The React UI. |
| `icons` / `action.default_icon` | `public/icon16.png`, `public/icon48.png`, `public/icon128.png` | Generated by `scripts/gen-icons.mjs`. |

### OAuth2

```json
"oauth2": {
  "client_id": "<redacted>",
  "scopes": [
    "https://www.googleapis.com/auth/drive.appdata",
    "https://www.googleapis.com/auth/userinfo.email",
    "https://www.googleapis.com/auth/userinfo.profile"
  ]
}
```

See [OAuth scopes](#oauth-scopes) below for what each scope grants.

### Content security policy

```json
"content_security_policy": {
  "extension_pages": "script-src 'self' 'wasm-unsafe-eval'; object-src 'self'"
}
```

`wasm-unsafe-eval` is required because PGlite is WebAssembly and must execute in the extension (offscreen) context. Without it, PGlite fails to load. See [Debugging](../how-to-contribute/debugging.md).

### Externally connectable

```json
"externally_connectable": { "matches": ["https://*.chromiumapp.org/*"] }
```

Used by the `launchWebAuthFlow` redirect flow.

## .env files

There are no `.env` files in this repository. The Chrome extension does not use `import.meta.env` for secrets; the OAuth `client_id` lives in `manifest.json`, and all other configuration is read from `chrome.storage.local` at runtime. The Vite `define` block in `chrome-extension/vite.config.ts` only stubs the `process` global for the browser build (see [Tooling](../how-to-contribute/tooling.md)); it does not inject environment variables.

If you need to swap the OAuth client (for example, for a staging build), edit `manifest.json`'s `oauth2.client_id` directly. There is no environment-based override.

## chrome.storage.local keys

The extension persists all runtime configuration and sync state in `chrome.storage.local`. The keys, where they are read and written, and what they hold:

| Key | Type | Written by | Read by | Purpose |
|---|---|---|---|---|
| `vaultSalt` | `number[]` | Settings view (set password) | `service-worker/index.ts` (`handlePush`, `handlePull`), crypto module | The PBKDF2 salt for the vault key. Presence indicates the vault has been initialised. |
| `autoLockMs` | `number` | Settings view | `sidepanel/App.tsx`, `sidepanel/views/SettingsView.tsx` | Idle lock timeout in milliseconds. When the side panel is idle for this long, the vault relocks. |
| `driveConnected` | `boolean` | `service-worker/index.ts` on connect | `service-worker/index.ts` (`handlePush`), `sidepanel/views/SyncView.tsx` | Whether a Google Drive account is connected. |
| `driveFileId` | `string` | `service-worker/index.ts` on first push | `service-worker/index.ts` (`getFileId`) | The Drive `appdata` file id used for the encrypted backup. |
| `syncedAt` | `string` (ISO date) | `service-worker/index.ts` on push/pull/import | `service-worker/index.ts`, `sidepanel/views/SyncView.tsx` | Timestamp of the last successful sync. |
| `driveEmail` | `string` \| `null` | `service-worker/index.ts` on connect | `sidepanel/views/SyncView.tsx` | The connected Google account's email. |
| `driveAvatar` | `string` \| `null` | `service-worker/index.ts` on connect | `sidepanel/views/SyncView.tsx` | The connected Google account's profile picture URL. |

On disconnect, `SyncView.tsx` removes `driveConnected`, `driveFileId`, `driveEmail`, and `driveAvatar` together via `chrome.storage.local.remove([...])`. `syncedAt` and `vaultSalt` are not cleared on disconnect.

## Android SharedPreferences

The Android sync layer stores its state in a single `SharedPreferences` file named `myspace_sync`, opened with `Context.MODE_PRIVATE`. It is accessed in two places:

- `android/app/src/main/java/com/myspace/app/sync/DriveRepository.kt` - `context.getSharedPreferences("myspace_sync", Context.MODE_PRIVATE)`
- `android/app/src/main/java/com/myspace/app/ui/screens/SyncScreen.kt` - same call, used to read/clear the token for the UI

Known keys:

| Key | Type | Purpose |
|---|---|---|
| `access_token` | `String?` | The Google OAuth access token used by `DriveRepository` for Drive API calls. Cleared on disconnect via `prefs.edit().remove("access_token").apply()`. |
| `drive_file_id` | `String?` | The Drive `appdata` file id, mirroring `driveFileId` from the extension. Read via `fileId()`, written via `setFileId(id)`. |

Sensitive values (the access token, stored secrets) should be routed through `androidx.security:security-crypto` EncryptedSharedPreferences rather than plain `SharedPreferences` when hardening is added; see [Dependencies](./dependencies.md).

## OAuth scopes

The same three Google OAuth scopes are requested by both the Chrome extension (in `manifest.json`) and the Android app (via the Credentials API). They are the minimal set needed for the encrypted backup feature:

| Scope | What it grants | Used for |
|---|---|---|
| `https://www.googleapis.com/auth/drive.appdata` | Read/write to the app-specific `appdata` folder in Drive (not visible to the user in their Drive UI). | Storing the single encrypted backup file. |
| `https://www.googleapis.com/auth/userinfo.email` | The user's email address. | Showing the connected account email in the Sync view. |
| `https://www.googleapis.com/auth/userinfo.profile` | The user's basic profile (name, profile photo). | Showing the Google avatar in the Sync view. |

No scope gives access to the user's general Drive files. The backup lives entirely in the hidden `appdata` namespace.
